# City_Quest_Explorer_Growth_Hacking - Growth Hacker (1000 usuarios)

Este archivo registra estrategias y estado del proceso de **Growth** para alcanzar los primeros **1000 usuarios** en **City Quest Explorer**.

## Objetivo
Conseguir los primeros 1000 usuarios.

## Especialidades
- TikTok
- Instagram Reels
- SEO Local
- Turismo
- Marketing de guerrilla

## Generar
- Campañas
- Embudos
- Estrategias virales
- Programas de referidos
- Lanzamientos

## Métricas (para controlar el embudo)
- [ ] Tráfico (views / sessions)
- [ ] Conversión (landing → registro)
- [ ] Activación (primer juego/misión iniciada)
- [ ] Retención (D1 / D7)
- [ ] Referidos (ratio y conversion de referidos)
- [ ] Coste por usuario (CPI / CPA si aplica)

## Embudo base (recomendado)
- Hook (video corto)
- Landing (SEO + ads orgánicas)
- Registro rápido
- Activación (primer “tour” o misión)
- Compartir resultado (social proof)
- Referidos (código/URL)

## Estado del proceso
- [ ] Definir ICP (ciudad, perfil, motivadores)
- [ ] Definir oferta inicial (gratis + prueba piloto por ciudad)
- [ ] Diseñar “viral loop” (logros/recompensas compartibles)

## Plan de campañas
- [ ] Campaña TikTok (series semanal)
- [ ] Campaña Instagram Reels (recap + UGC)
- [ ] SEO Local (landing por ciudad + schema)
- [ ] Marketing de guerrilla (activaciones en puntos turísticos)

## Programa de referidos
- [ ] Definir recompensa para invitante y referido
- [ ] Definir tracking (código único/UTM)
- [ ] Definir anti-fraude (límites y validación básica)

## Lanzamientos

- [ ] Lanzamiento “Misión Especial” (campaña limitada)

## Checklist de documentación (por cada campaña)
- [ ] Hipótesis (qué creemos que funcionará y por qué)
- [ ] Creativos (guiones / hooks / CTA)
- [ ] Distribución (canales + calendario)

- [ ] Resultado (KPI) + aprendizajes + iteración
